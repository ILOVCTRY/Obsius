### PSK
**载波相位变化**：让高频正弦波"突然跳一个角度"来表示不同的数字。
```tex
时间 →

正常载波:  ~  ~  ~  ~  ~  ~  ~
            ↑   ↑   ↑
            波峰位置固定

相位突变:   ~  ~  \  /  ~  ~  \  /
                ↑
                这里波峰突然"跳"了一下
```
PSK：Phase Shift Keying，**相移键控**）是一种用**载波相位变化**来传递数字信息的调制方式。

| 发送的比特 | 相位角度 | 直观描述     |
| ----- | ---- | -------- |
| 000   | 0°   | 正常出发，不转弯 |
| 001   | 45°  | 向右转一点    |
| 010   | 90°  | 直直向上     |
| 011   | 135° | 向左上      |
| 100   | 180° | 完全掉头     |
| ...   | ...  | ...      |
| 111   | 315° | 向右下      |
解析信号：
如果实信号是 x(t) ，它的解析信号 z(t) 定义为：
	$\hat{x}(t) = \mathcal{H}\{x(t)\}$,$z(t)=x(t)+j\cdot \hat{x}(t)$

其中：
- x(t) 是原始实信号
- $\hat{x}(t)$ 是 x(t) 的**希尔伯特变换**
- j 是虚数单位

z(t) 是一个**复数信号**，每个采样点都可以画在复平面上：
```tex
        Im(虚部)
          ↑
          |    ● z(t) = a + jb
          |   /|
          |  / |  幅度 = |z| = √(a²+b²)
          | /  |  相位 = ∠z = atan2(b, a)
          |/θ  |
    ------+----+----→ Re(实部)
```
- **瞬时幅度**：$∣z(t)∣=\sqrt{x^2+\hat{x}^2}​$（包络）
- **瞬时相位**：$∠z(t)=atan2(\hat{x},x)$，复角 = 复数在复平面上的"方向角"，`np.angle()` 就是提取这个角度，也就是信号的瞬时相位。


验证是否为PSK的代码：

```python
from scipy.io import wavfile      # 导入读取 WAV 音频文件的函数
from scipy.signal import hilbert  # 导入希尔伯特变换，用于构造解析信号
import numpy as np                # 导入 NumPy 用于数值计算

# ==================== 1. 读取音频信号 ====================
rate, data = wavfile.read(r'D:\Dowload\codex_reverse\flag.wav')  
# 读取 WAV，rate 为采样率，data 为采样点数组

# ==================== 2. 提取瞬时相位 ====================
ana = hilbert(data)     # 对实信号做希尔伯特变换，得到复解析信号（含实部+虚部）
phase = np.unwrap(np.angle(ana))   
# 先取复角得到瞬时相位（范围 [-π, π]），再用 unwrap 解卷绕消除 ±π 跳变，拉成连续曲线

# ==================== 3. 盲估计载波频率 ====================
t = np.arange(len(data)) / rate 
# 构造时间轴，单位秒：[0, 1/rate, 2/rate, ..., (N-1)/rate]

coef = np.polyfit(t, phase, 1)  #t是时间轴。phase是瞬时相位
# 对相位-时间做一阶线性拟合，斜率即 2π*载波频率，截距为初始相位

carrier_hz = coef[0] / (2 * np.pi)       
# 由斜率换算出载波频率（Hz），因为 d(phase)/dt = 2π * f_c

print('carrier_hz', carrier_hz)     # 打印估计出的载波中心频率

# ==================== 4. 去除载波，提取纯调制相位 ====================
resid = phase - np.polyval(coef, t)      # 用拟合出的载波相位趋势减去总相位，剩余 resid 即为调制引起的相位变化（已去载波）
print('resid_minmax', float(resid.min()), float(resid.max()))  # 打印残差的最小、最大值，观察相位波动范围
print('resid_std', float(resid.std()))   # 打印残差标准差，衡量相位离散程度

# ==================== 5. 分窗检测 8-PSK 符号 ====================
for n in [147, 294, 441]:                # 遍历三种窗长：147(1符号)、294(2符号)、441(3符号)，验证哪个窗长下相位最聚集
    valid_len = (len(resid) // n) * n    # 计算能被 n 整除的最大长度，避免尾部截断
    sy = resid[:valid_len].reshape(-1, n)  # 将残差序列按每 n 点一行进行 reshape，每行代表一个窗（潜在符号周期）
    means = sy.mean(axis=1)              # 对每个窗沿时间轴（列方向）求平均，得到该窗的平均相位（抑制噪声）

    # 将平均相位映射到 8-PSK 的 8 个扇区（0~7），每个扇区宽 45°（2π/8）
    normalized = (means % (2 * np.pi)) / (2 * np.pi)  # 先把相位折叠到 [0, 2π)，再归一化到 [0, 1)
    q = np.round(normalized * 8) % 8                  # 乘以 8 映射到 [0,8)，四舍五入后取模 8，得到整数扇区编号 0~7

    # 统计各扇区出现的频次，验证是否均匀分布着 8 个离散值
    vals, counts = np.unique(q.astype(int), return_counts=True)  # 提取唯一的扇区编号及其出现次数
    print('n', n, list(zip(vals.tolist(), counts.tolist()))[:10])  # 打印窗长和对应的 (扇区号, 计数) 列表
```

### Raw 无线电 AFSK1200

AX.25 协议：**AX.25** 是业余无线电（Ham Radio）常用的数据链路层协议。它主要用于 **包通信（packet radio）**，比如通过无线电发送文本消息或遥测数据。- 协议定义了帧结构，但它的数据是通过 **音频信号调制到无线电频率** 传输的。

BFSK 调制：BFSK = Binary Frequency Shift Keying（二进制频移键控）- 这种调制的原理是：
    - **1 个比特 = 一个频率**
    - **0 个比特 = 另一个频率**
- 换句话说，信号的幅度不变，但频率在两个值之间跳动来表示 0 和 1。


故当我们发现，在 xxxx Hz 和 yyyy Hz 处有两个峰值，这是 AX.25 的 BFSK 中使用的两个音调。
![image.png](https://raw.githubusercontent.com/ILOVCTRY/note-gen-image-sync/main/blog-img20260502134133031.png)

需要先用 `sox` 把 wav 转为 raw

```bash
sox -t wav 997a0b28705f4ef086acfb7e1b932336 -esigned-integer -b16 -r 22050 -t raw latlong.raw

# sox: 调用 SoX 音频处理工具
# -t wav: 指定输入文件的类型为 WAV 格式
# 997a0b28705f4ef086acfb7e1b932336: 输入文件名
# -esigned-integer: 指定输出音频的编码格式为有符号整数
# -b16: 指定输出音频的位深为 16 位（即每个样本占 16 位）
# -r 22050: 指定输出音频的采样率为 22050 Hz
# -t raw: 指定输出文件的类型为 RAW（原始音频数据，无文件头）
# latlong.raw: 输出文件名
```

再使用工具 `multimon-ng` 解密
```bash
multimon-ng -t raw -a AFSK1200 latlong.raw

# -t 指定输入文件的类型
# -a 指定要使用的解码协议
# AFSK1200 表示解码 1200 bps 的音频频移键控（Audio Frequency-Shift Keying） 信号，这是 APRS（自动分组报告系统）等协议常用的调制方式
```
### lsb隐写

```python
import wave, numpy as np
p=r'D:\Dowload\codex_reverse\temp\drum_bits\drum_bits.wav'
with wave.open(p,'rb') as w:
    frames=w.readframes(w.getnframes())
    x=np.frombuffer(frames,dtype='<i2')  //把音频字节转成数字数组
print('samples', x.size)  //音频里有多少个数字（采样点）
print('min', int(x.min()), 'max', int(x.max()))//声音波形的范围
print('mean', float(x.mean()))    //声音的平均强度
print('std', float(x.std()))      //平均波动大小
print('zero_frac', float(np.mean(x==0)))  //采样值等于 0 的比例
print('lsb0_frac', float(np.mean((x&1)==0)))   //最低位是 0 的比例
print('lsb1_frac', float(np.mean((x&1)==1)))   //最低位是 1 的比例
```

正常声音：
- LSB 是 0 和 1 **随机分布**（各 50% 左右）
如果有人藏了秘密数据：
- LSB 会被改成 0/1 来存储信息
- 比例会**明显偏离 50%**

```tex
samples 1210778
min -32431 max 28698
mean 143.01446425356258
std 5167.023397868183
zero_frac 0.6599913444083061    //66% 的采样值是 0（异常高，静音区很多）
lsb0_frac 0.8263191105223253    //82.6% 的最低位是 0
lsb1_frac 0.17368088947767468   //17.4% 的最低位是 1
```
说明隐藏了数据，8bits为一字节。

```python
import wave, numpy as np

# 你的音频路径
p = r'D:\Dowload\codex_reverse\temp\drum_bits\drum_bits.wav'

# 读取音频采样
with wave.open(p, 'rb') as w:
    frames = w.readframes(w.getnframes())
    x = np.frombuffer(frames, dtype='<i2')

# 提取所有采样的【最低位】(LSB)
bits = x & 1

# 把 8 个 bit 拼成 1 个字节
output = []
for i in range(0, len(bits), 8):
    byte = 0
    for j in range(8):
        if i + j >= len(bits):
            break
        byte = (byte << 1) | bits[i + j]
    output.append(byte)

# 转成bytes并打印（能直接看到隐藏内容）
hidden_data = bytes(output)

print("==== 提取到的隐藏数据 ====")
print(hidden_data)
print("\n==== 尝试解码为字符串 ====")
print(hidden_data.decode("utf-8", errors="replace"))
```
