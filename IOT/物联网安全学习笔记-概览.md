## 二、IoT 设备信息收集

### 1. 端口扫描

使用 Nmap 识别设备开放端口、服务类型和潜在攻击面。

常规扫描：

```bash
nmap -Pn 192.168.0.106
```

`-Pn`：跳过主机发现，直接将目标视为在线。

UDP 服务扫描：

```bash
nmap -sV -sU -Pn --open 192.168.0.1
```

参数含义：

- `-sU`：UDP 端口扫描
- `-sV`：识别服务和版本
- `-Pn`：跳过主机发现
- `--open`：只显示开放或可能开放的端口

曾记录的开放端口示例：

- `80/tcp`：HTTP 服务
- `554/tcp`：RTSP 服务
- `9898/tcp`：Nmap 识别为 `monkeycom`，实际服务需要继续验证

### 2. 流量抓取

- 外部嗅探
- 设备联网抓包
- 内部调试抓取
- 通过串口观察设备通信和调试信息

### 3. 功能点评估

梳理设备提供的：

- 管理功能

- 识别固件结构

- 仿真处理器架构

# 五、物联网设备常见漏洞点

## 1. 路由器后台功能漏洞

### 网络诊断命令注入

路由器后台的 Ping、Traceroute、DNS 查询等网络诊断功能，可能将用户输入直接拼接到系统命令中，从而造成：

- 命令注入
- 远程命令执行
- 设备权限失陷

重点关注：

```text
用户输入
→ 参数解析
→ 字符串拼接
→ system()/popen()
→ 命令执行
```

### 配置文件导出

可能造成：

- 管理员密码泄露
- Wi-Fi 密码泄露
- 密钥和证书泄露
- 网络配置泄露
- 敏感系统信息泄露

### 配置文件导入

可能存在：

- 恶意配置注入
- 配置篡改
- 文件解析漏洞
- 路径处理漏洞
- 命令注入
- 任意文件覆盖

---

## 2. Web 管理端漏洞

### 认证绕过

敏感路径在未登录状态下可直接访问，例如：

```text
view.html
system.html
```

需要检查：

- 页面是否鉴权
- 接口是否鉴权
- 前端限制是否可绕过
- 后端是否独立校验身份

### Cookie 伪造

常见原因：

- Cookie 内容可预测
- Cookie 可直接修改
- 会话签名缺失
- 服务端校验不足
- 用户权限直接保存在客户端 Cookie 中

### 弱密码

包括：

- 默认密码
- 简单密码
- 固定密码
- 不同设备共用密码
- 硬编码账号密码

# 七、RTSP 服务安全

## RTSP 未授权访问

RTSP 常用端口：

```text
554
```

如果设备未启用认证或访问控制不严，攻击者可能直接查看实时视频流。

### RTSP 路径获取方式

通过固件逆向分析，在以下位置查找：

- Web 前端脚本
- 配置文件
- 可执行程序
- 启动脚本
- 默认配置

常用搜索关键词：

```text
rtsp://
554
live
stream
channel
main
sub
```

也可以查询：

- 厂商说明书
- SDK
- 开发文档
- 技术论坛
- 公开漏洞资料
- 厂商常见 RTSP URL 规则

测试时关注：

- 是否要求账号密码
- 是否存在默认凭据
- RTSP 路径是否固定
- 通道编号规则
- 主码流和辅码流
- URL 参数规则

# 八、基于固件二进制的漏洞风险

## 1. 栈溢出与堆溢出

常见原因：

- 固定长度缓冲区
- 输入长度校验缺失
- 不安全字符串复制
- 超长 HTTP 参数或请求头
- 协议字段长度未校验

常见危险函数：

```c
strcpy()
strcat()
sprintf()
gets()
memcpy()
```

### D-Link DIR-815 案例

- 可控输入字段：Cookie
- 向 Cookie 输入超长数据
- 可能导致栈缓冲区溢出

分析重点：

```text
HTTP Cookie
→ 请求解析
→ 字符串处理
→ 固定大小栈缓冲区
→ 栈溢出或程序崩溃
```

### Cisco RV110W 案例

记录为栈溢出相关案例。

尚待补充：

- 触发路径
- 可控参数
- 影响版本
- CVE 编号
- 漏洞函数

---

## 2. 拒绝服务

可能表现为：

- 程序崩溃
- 服务停止
- 服务重启
- 设备重启
- 死循环
- 内存或 CPU 资源耗尽

### 空指针崩溃

程序处理异常输入或错误状态时，没有判断指针是否为空就直接解引用，造成：

- 进程崩溃
- 服务不可用
- 设备功能中断
- 持续触发形成拒绝服务

重点检查：

- 空指针判断
- 函数返回值
- 错误处理分支
- 缺失字段
- 异常协议报文
- 外部输入是否可控

---

## 3. 整数溢出

可能造成：

- 长度计算错误
- 内存分配不足
- 边界检查绕过
- 越界读写
- 程序崩溃

---

## 4. 越界读写

### 越界读取

可能导致：

- 内存数据泄露
- 密钥或凭据泄露
- 地址信息泄露
- 程序崩溃

### 越界写入

可能导致：

- 数据破坏
- 程序崩溃
- 控制流劫持
- 代码执行

---

## 5. 命令注入

固件二进制将外部输入拼接到系统命令中，例如调用：

```c
system()
popen()
```

常见输入来源：

- Web 参数
- Cookie
- HTTP 请求头
- 配置文件
- 协议字段
- 网络诊断参数
- 设备名称
- 文件名

# 九、后门漏洞

## 调试后门

设备正式固件中保留开发或测试功能，例如：

- 隐藏调试账号
- 默认账号密码
- 硬编码凭据
- Telnet 服务
- SSH 调试服务
- 隐藏管理页面
- 特殊请求参数
- 魔术字符串
- 未公开命令
- 测试接口
- 后台诊断功能

可能造成：

- 绕过认证
- 获取管理员权限
- 执行系统命令
- 访问敏感数据
- 长期控制设备

# 十、逻辑漏洞

## 1. 任意文件读取

文件查看、下载、日志读取或配置导出功能未严格限制文件路径，攻击者可能读取设备任意文件。

可能泄露：

- `/etc/passwd`
- `/etc/shadow`
- 配置文件
- 管理密码
- Wi-Fi 密码
- 密钥
- 证书
- 日志
- 启动脚本

## 2. 路径穿越

常见形式：

```text
../
../../
绝对路径
URL 编码后的 ../
双重编码
混合路径分隔符
```

本质是绕过预期目录限制，访问目录外文件。

## 3. 任意密码重置

密码重置流程中缺少身份验证或权限校验，例如：

- 不验证旧密码
- 验证码可预测
- Token 可伪造
- Token 不绑定用户
- 可修改用户 ID
- 接口未鉴权
- 普通用户可重置管理员密码

## 4. 符号链接漏洞

程序处理以下功能时没有安全检查符号链接：

- 临时文件
- 日志文件
- 配置导入导出
- 备份恢复
- 固件升级
- 文件上传
- 文件覆盖

可能导致：

- 任意文件读取
- 任意文件覆盖
- 越权修改系统文件

# 十一、云端安全风险

## 1. 传统 Web 安全风险

### SQL 注入

常见位置：

- 登录接口
- 用户查询
- 设备查询
- 日志检索
- 数据统计
- 设备管理接口

可能造成：

- 认证绕过
- 数据泄露
- 数据篡改
- 删除数据
- 数据库权限失陷

### 命令注入

云端接口将用户输入拼接到系统命令中，可能造成：

- 远程命令执行
- 云端服务器失陷
- 敏感数据泄露
- 横向移动
- 控制已接入的 IoT 设备

## 2. 越权访问

### 水平越权

普通用户通过修改以下标识，访问其他用户的资源：

- 用户 ID
- 设备 ID
- 日志 ID
- 家庭 ID
- 文件 ID
- 订单号

### 垂直越权

低权限用户调用管理员接口或执行高权限操作。

测试时不能只检查对象是否存在，还要检查：

```text
当前用户是否拥有该对象
当前用户是否具备执行该操作的权限
```

# 十二、模糊测试

## 1. 端口与网络服务 Fuzz

针对开放端口对应服务发送：

- 超长数据
- 空数据
- 随机数据
- 畸形数据
- 边界值
- 非法字段
- 异常消息顺序

观察：

- 服务崩溃
- 设备重启
- 连接断开
- 异常响应
- 超时
- CPU 或内存占用异常

## 2. Burp Suite Fuzz

通过 Burp Intruder 对以下位置进行批量变异：

- URL 路径
- 查询参数
- Cookie
- HTTP 请求头
- 表单字段
- JSON 字段
- 文件名
- 数值字段

## 3. 协议 Fuzz

常用工具：

- boofuzz
- Peach

重点测试：

- 字段长度
- 字段类型
- 数值边界
- 校验值
- 消息顺序
- 协议状态机
- 长度字段与实际数据不一致
- 重复消息
- 缺失字段
