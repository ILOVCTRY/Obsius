## 1. 车联网安全学习的基本思路

车联网安全并不是单独研究 CAN 总线，而是传统网络安全、移动安全、嵌入式安全、无线通信和汽车电子的交叉方向。

一条典型的车联网攻击路径可能经过：

```text
汽车 APP / 云端平台
        ↓
HTTPS、MQTT 或私有协议
        ↓
T-BOX 或 IVI 车机
        ↓
中央网关
        ↓
CAN、车载以太网、UDS
        ↓
BCM、VCU、BMS 等 ECU
```

因此，学习时应同时关注以下几个层面：

- 云端 API 和远程控制平台；
- 汽车移动 APP；
- IVI 智能座舱和 T-BOX；
- Wi-Fi、蓝牙、NFC 等无线入口；
- CAN、CAN FD、LIN 和车载以太网；
- ISO-TP、UDS、DoIP、SOME/IP；
- ECU 固件、硬件接口和 OTA；
- 整车网络架构、网关隔离和风险评估。

这篇文章最重要的价值，是说明汽车安全比赛并不只考 CAN 和 UDS。很多题目仍然基于传统 Web、密码、逆向和 Pwn，只是将数据对象换成 VIN、GPS、OTA、车辆凭据或车联网通信。

其中最值得车联网方向重点阅读的是：

- **VIN**：展示“蓝牙流量—APK—JNI—SO—车辆数据”的分析链路；
- **GPS**：展示车辆轨迹数据分析；
- **OTA-流量**：展示升级会话、车辆状态和密钥数据分析；
- **ezapk**：展示汽车应用中的 Native 校验和本地凭据风险。

如果以实车比赛为目标，可优先阅读这四道题；如果以线上赛晋级为目标，则需要同时补齐 Crypto、Web、Reverse 和 Pwn。

## 12. UDS 刷写安全

- [车联网安全基础知识之 UDS 刷写安全](https://www.iotsec-zone.com/article/393)

这篇文章主要介绍 ECU 刷写流程：

```text
读取版本信息
  ↓
切换诊断会话
  ↓
SecurityAccess
  ↓
RequestDownload
  ↓
TransferData
  ↓
完整性检查
  ↓
ECU Reset
```

阅读重点包括：

- Seed-Key 是否过弱；
- 固件是否签名；
- 是否存在回滚保护；
- 是否验证目标 ECU；
- 是否记录刷写工具和操作信息；
- 中断刷写是否会导致 ECU 失效。

它可以与 CCF IVC 的 OTA 流量题配合阅读，分别理解“云端升级包下载”和“车端 ECU 刷写”。

## 20. 两篇经典汽车安全论文

- [论文入口](https://checkoway.net/papers/car2010/)

- 外部通信组件；

5. T-BOX、IVI 和中央网关；
6. Web/API 到 CAN/UDS 的综合模拟环境。

可以搭建如下本地练习链：

```text
Web 或模拟 APP
        ↓
API / MQTT
        ↓
Python 车联网网关
        ↓
vcan0
        ↓
ICSim 或 UDSCTF
```

这样可以把云端、车端和车内协议串联起来，比单独练习 CAN 或 UDS 更接近真实车联网架构。

# 总结

这份资料可以作为车联网安全的入门和赛事参考目录。

建议按照以下逻辑学习：

```text
赛事复盘
  ↓
汽车组件和网络架构
  ↓
CAN、ISO-TP、UDS
  ↓
APP、IVI、T-BOX 和云端
  ↓
OTA、固件和整车攻击链
  ↓
标准、风险和安全工程
```

对于已经完成 ICSim、UDSCTF 和 crAPI 的学习者，下一阶段应重点关注：

- IVI 和 Android 车机；
- MQTT 与车辆服务；
- T-BOX 和中央网关；
- OTA 与 ECU 刷写；
- 云端 API 到 CAN/UDS 的跨层攻击链。
