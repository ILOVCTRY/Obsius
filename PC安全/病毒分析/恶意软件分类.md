# 恶意软件分类

## 病毒

### 病毒简介

- 如果恶意代码将其自身的副本添加到文件、文档或磁盘驱动器的启动扇区来进行复制，则被认为是病毒。
- 不主动窃取数据、不远程控制，核心危害是感染文件、破坏系统。
- **必须依赖宿主文件**，依附 EXE/DLL/ 文档存在，核心特性是**自我复制感染**，无自主传播能力，需要用户主动运行宿主程序。

### 病毒携带者

- 可执行文件：`.exe, .com, .sys, .dll, .ocx, .prg`
- 脚本：`.vbs, .js, .wsh, .prl`
- 宏：宏脚本语言，如Office文档
- 启动扇区：启动扇区是存储设备的第一个扇区通常是512字节，也是CPU开机或引导首先加载并执行的代码。

### 病毒的分类

#### 按照链接方式分类

- 源码型病毒：通过感染源代码文件（如，`.c, .cpp, .py`等），而非感染编译后的二进制文件。
	- 工作方式如下：
	  1. 病毒扫描文件系统，找到目标源代码文件
	  2. 将自身恶意代码插入到源代码中
	  3. 当源代码被编译或者执行时，病毒代码也随之执行
	  4. 执行后再次扫描并感染更多源码文件。

- 嵌入型病毒：其将自己的恶意代码嵌入到宿主文件内部（`EXE, DLL, SCR, SYS`等），宿主正常执行时先触发病毒代码
	- 嵌入的主要方式：
		1. 前置：病毒代码插在宿主文件头部，宿主被后移。
		2. 后置：病毒追加到文件末尾，修改入口点指向病毒。
		3. 间隙插入：植入PE文件节对齐的空白区域。

- 外壳病毒：不修改宿主文件代码，而是把宿主文件包在自己里面，病毒代码充当外壳，执行时先跑病毒。
	- 打包器：
	  1. 如UPX, ASPack这些。

- 译码型病毒：使用**加密/编码**保护自身代码，运行时通过一个简短的**解密器（decryptor stub）** 在内存中解码后执行。
	- 工作方式：
	  1. 文件中的病毒（加密/编码，不可直接分析）
	  2. 执行解密器
	  3. 解密器在内存中解密出完整的病毒代码
	  4. 跳转到解密的病毒执行。
	- 特点：
		- 免杀：静态扫描没用，特征码失效。
		- 多态变异：每次感染生成不同的解密器+不同密钥，无固定特征。

- 操作系统型病毒：将恶意代码植入**操作系统核心组件**（如 `ntoskrnl.exe`、`hal.dll`、系统服务驱动、设备驱动程序），在系统启动或运行时获得最高控制权。
	- 常见感染目标：
	  1. 内核文件：替换或修改`ntoskrnl.exe, kernel32.cll, ntdll.dll`。
	  2. 系统驱动：注入`.sys`驱动
	  3. 引导组件：MBR，VBR，bootmgr，NTLDR
	  4. 系统服务

#### 按照病毒存储方式

- 引导型病毒：在系统启动**最早期**、操作系统加载之前获得控制权。
	- 工作流程：
	  1. 开机->BIOS->读 MBR/VBR
	  2. 引导病毒代码先执行替换了原来的引导代码
	  3. 病毒驻留内存，通常是hook int 13h 磁盘中断
	  4. 病毒加载真正的 MBR/VBR ->正常启动系统

- 文件型病毒：通过感染可执行文件如`exe, com, dll, sys, scr`等，在宿主程序执行时获得控制权。
	- 感染方式：[参考嵌入式病毒](PC安全/病毒分析/恶意软件分类.md#^704303)

- 目录型病毒：不感染文件内容，而是感染文件系统的目录入口/目录项，利用操作系统读取目录的逻辑来执行恶意代码。
	- 工作方式：
	  1. 用户双击`exe`文件
	  2. 操作系统查找目录表->找到该`exe`文件的起始簇号。
	  3. 目录项已经被修改，指向病毒代码所在的簇。
	  4. 病毒先执行，再跳回原来的簇。

- 混合型病毒：同时使用两种或以上方式进行传播。
	- 常见组合：
	  1. 引导型+文档型
	  2. 操作系统型+文件型
	  3. 引导型+操作系统型

### 病毒的命名

- 三元组：前缀 + 病毒名 + 后缀
	- 前缀：表示该病毒发生的操作系统平台或者病毒的类型。
	- 病毒名：该病毒的家族
	- 后缀：一般不需要，主要区别病毒家族中的各病毒的不同。


### 计算机病毒的逻辑结构

- 感染标志
	- 有的病毒有一个感染标志，又称病毒签名。
	- 感染标志一般是一些数字或者字符串，以ASCII的形式存储在宿主文件中。
	- 病毒在感染前，一般要查看是否程序已有感染标志。
	- 可以提取感染标志作为特征码。
	- 也可以根据这个特性，人为添加感染标志，来避免病毒感染。

- 引导模块：
	- 染毒程序运行时，会先运行病毒的引导模块。
	- 引导模块的基本动作是：
		- 检查运行的环境：如确定操作系统类型，内存容量，现行区段，磁盘设置，显示器类型等。
		- 将病毒引入内存，使病毒处于动态，并保护内存中的病毒代码不被覆盖。
		- 设置病毒的激活条件和触发条件，触发后调用感染模块或者破坏模块。

- 感染模块：
	- 寻找感染目标
	- 检查目标是否满足感染条件，再决定是否感染。

- 破坏模块：这个就不必多说了，下面简单补充一点内容。
	- 触发条件：多种多样，如日期，特定键盘输入等

- 传染方式：所谓传染，指的是计算机病毒由一个载体传递到另一个载体，由一个系统传递到另一个系统。
	- 被动传染：用户在拷贝或者传输时，把一个病毒复制到另一个载体或者系统中。
	- 主动传染：病毒自动的借助宿主机，向其他载体或者系统扩散。

## 蠕虫

- **独立恶意程序，无需宿主**，具备**主动网络传播能力**，利用漏洞、弱口令横向扩散。
- 自愈性强、传播速度快，核心危害是占用网络带宽、批量感染主机、构建僵尸网络

## 木马

- 伪装成正常软件，**无自我复制能力**，核心目的是**窃取信息、植入后门、为攻击者提供入口**。
- 病毒和蠕虫会将木马作为攻击载荷的一部分复制到目标系统上。
- 木马入侵宿主机后允许攻击者获取宿主机的远程访问权限。

### 木马的结构

木马通常采用服务端、客户端结构。功能上由木马配置程序、控制程序和木马程序构成。

- 木马程序：又被称为服务器程序，负责获取远程权限，和接收控制指令。
- 木马配置程序：木马配置程序设置木马程序的端口号、触发条件、木马名称等，使其在服务端隐藏更好。
- 控制程序：控制程序控制远程木马服务器，统称为控制端程序，负责配置服务器、给服务器发送指令。

### 木马的分类

#### 特洛伊木马

- 远程访问特洛伊：某些木马程序使攻击者可以通过远程控制系统，此类程序称为“远程访问特洛伊”

#### Rootkit

- Rootkit属于内核后门病毒，对操作系统内核进行劫持，通过隐藏其他病毒进程、注册表、文件相 关操作等方式与安全软件进行对抗。

#### BootKit

- 这个是更高级的Rootkit，通过感染MBR（磁盘主引记录），实现绕过内核检查和启动隐身。属于引导后门病毒。
	- 广义：所有在开机时比Windows内核更早加载，实现内核劫持的技术。
	- 例如：[BIOS Rootkit](#BIOS%20Rootkit)、[Vbootkit](#Vbootkit)、[SMM Rootkit](#SMM%20Rootkit)等

- 其具有以下特点：
	- 在Ring3下可完成Hook（改写）
	- 注入内核的代码没有内存大小限制，也无需自己读入代码。
	- BootDriver驱动初始化时加载(依情况而定，也可hook内核其它地方)
	- 理论上可以Hook各种版本ntldr
	- 理论上可以引导各个版本NT内核和内存相关boot.ini参数

##### BIOS Rootkit

BIOS Rootkit 是感染计算机**固件层**（BIOS/UEFI）的恶意代码，位于硬盘之前、操作系统之下。
核心特点有：
-  **持久性极强**：重装系统、换硬盘、格式化都无法清除，它驻留在主板 SPI Flash 芯片中
- **先于 OS 加载**：BIOS 初始化时即执行，可以 Patch OS 内核或禁用安全机制（如 Secure Boot、HVCI）
-  **隐形**：操作系统层面的杀毒软件和 EDR 看不到它，因为它运行在 **SMM（System Management Mode）** 中，对 OS 完全透明


##### Vbootkit

**Vbootkit** 是专门针对 **Windows Vista/7 及之后版本**的 bootkit，在操作系统内核加载过程中劫持启动流程。

##### SMM Rootkit

**SMM Rootkit** 驻留在 **SMM（System Management Mode，系统管理模式）** 中，是 x86/x64 架构中**权限最高、最底层**的执行环境（比 Hypervisor 和 Ring 0 都低）。

- 集成（捆绑）到应用程序中。
- 隐藏在`Autoexec.bat`和`Config.sys`中。

## 逻辑炸弹

- 合法的应用程序，只是在编程时被故意写下某种“恶意功能”，在一定程度（如时间，次数，或者某种逻辑组合）时会触发。
- 例如，作为版权保护方案，某个应用程序有可能会在运行几次就在硬盘中将自己删除。

## 后门

- 后门是恶意攻击者选择用来远程连接系统的工具。典型的后门会在运行它的主机上打开一个网络端口，然后等待攻击者的远程连接。

## 勒索文件

核心逻辑：**文件加密 + 勒索赎金**，通过对称 + 非对称加密加密用户文档、数据，删除备份，弹窗索要比特币解密。

特点：无主动控制、极少窃取数据，核心危害是数据损毁、业务瘫痪。
